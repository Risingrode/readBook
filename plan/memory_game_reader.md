# 动感记忆阅读器 (Dynamic Memory Reader) 详细规划

## 1. 核心阅读模式设计

### A. PPT 幻灯片分块阅读 (Memory Slides)
- **机制:** 将长 TXT 文档自动切分为“记忆卡片”。
- **体验:** 每次只显示一屏。用户通过空格键或点击翻页。这种方式能强制大脑专注于当前这 1-2 个知识点，防止跳读。
- **动效:** 翻页时使用 3D 翻转或卡片堆叠动画，给用户“掌控感”。

### B. 动感文字 (Kinetic Typography)
- **文字乱跑 (Text Physics):** 重要概念或关键词可以设置为“物理物体”，在屏幕上轻微抖动、旋转或受重力下坠。
- **视觉追逐:** 系统自动高亮正在阅读的句子，周围的文字则呈半透明或轻微晃动。
- **动态阅读速度 (RSVP):** 引入“极速模式”，单词一个接一个地在中心闪现，速度可调。

## 2. “玩”与“记”的融合 (Learning Mini-Games)

### A. 文字捕捉 (Keyword Capture)
- 在阅读过程中，关键词会突然变色或试图“跑掉”。
- 用户必须点击它们（捕捉）来获得积分（EXP）。这种“手脑配合”能强力加深对关键词的记忆。

### B. 填空消消乐 (Memory Match/Fill)
- 在完成一个“卡片”后，系统自动隐藏 2-3 个核心词汇。
- 文字下方会出现打乱的候选词，用户拖拽正确的词放回去，文字会产生“融合”成功的发光效果。

### C. 进度领土化 (Map Conquest)
- 每读完一章，UI 界面会从灰白变为彩色，甚至长出一棵小树（知识之树）。

## 3. UI/UX 规划 (Interaction Design)
- **控制面板:** 悬浮在右侧，像游戏控制台一样，包含“重力开关”、“动画速度”、“记忆难度”调节器。
- **声音反馈:** 捕捉单词时有清脆的音效；完成卡片时有成就音。

## 4. 技术方案路线
- **动画库:** `framer-motion` (React 首选)。
- **物理模拟:** `matter-js` (如果需要极高质量的物理效果) 或 `react-use-gesture` 处理拖拽互动。
- **状态管理:** 使用 `Zustand` 实时追踪分数和连击（Combo）。
